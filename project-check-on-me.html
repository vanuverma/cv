Overview

Check On Me is a comprehensive emergency alert system designed to help users stay connected with their emergency contacts through automated wellness checks. If a user doesn't respond to a scheduled check-in, the system automatically notifies their designated emergency contacts.

The Problem

Many people, especially those living alone or with health conditions, need a reliable way to ensure someone knows if they're in trouble. Traditional methods like manual check-ins are unreliable and can create unnecessary worry.

The Solution

Check On Me automates the wellness check process with scheduled alerts, emergency contact management, and instant notifications across multiple channels (SMS, Email). The system includes offline support and intelligent retry logic to ensure reliability.

  3. Frontend Services (10s startup): Admin dashboard and mobile apps connect to the backend API

Technical Challenges & Solutions

Challenge: Reliable Background Task Scheduling

Problem: iOS and Android have different restrictions on background processes, making it difficult to ensure wellness checks run reliably.

Solution: Implemented a hybrid approach using workmanager for Android and local notifications with background fetch for iOS. Added server-side fallback to send alerts even if the app is closed.

Challenge: Scalable Notification System

Problem: Needed to send notifications to multiple contacts via different channels (SMS, Email) with retry logic for failures.

Solution: Built a job queue system using Bull and Redis. Each notification is a separate job with configurable retry attempts and exponential backoff. Priority queue ensures emergency alerts are processed first.

Challenge: Platform Compatibility

Problem: Some Flutter packages required newer iOS/Android versions, limiting potential users.

Solution: Replaced auth0_flutter with flutter_appauth, which supports iOS 12.0+. Carefully selected packages to maximize platform coverage (iOS 12.0+ and Android 6.0+), reaching 99% of active devices.

Challenge: Development Workflow

Problem: Managing three separate applications (mobile, backend, dashboard) with complex dependencies was time-consuming.

Solution: Created automated startup script (start-all.sh) that checks prerequisites, starts Docker containers, waits for services to be ready, and launches all components in the correct order with proper dependency handling.
